# Workflows Blueprint

## Role

This blueprint formalizes a team or agent workflow: phases, inputs, outputs, validations, and stop points.

## When To Use

- To describe a delivery flow.
- To standardize a review, test, or release process.
- To coordinate multiple agents without refactoring the kit.

## Expected Files

- `workflow.blueprint.yaml`: workflow contract.
- `workflow.template.md`: workflow template.

## Minimal Example

```text
Generate a release workflow with phases: preparation, verification, tag, PR.
```
