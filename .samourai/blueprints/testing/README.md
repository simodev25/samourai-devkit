# Testing Blueprint

## Role

This blueprint produces a clear test plan tied to acceptance criteria and risks.

## When To Use

- Before implementing a feature.
- To cover a bug fix.
- To document expected manual and automated tests.

## Expected Files

- `test.blueprint.yaml`: test-plan contract.
- `test-plan.template.md`: test-plan template.

## Minimal Example

```text
Generate a test plan for ticket GH-123 with unit and E2E tests.
```
