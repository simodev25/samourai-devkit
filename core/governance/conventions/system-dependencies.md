# System Dependencies

External tools required by Samourai Devkit install and uninstall scripts.

This document covers the shell programs shipped by the devkit:

| Program | Path | Purpose |
|---------|------|---------|
| `install-samourai.sh` | `scripts/install-samourai.sh` | Install Samourai core and selected editor adapters into a target project |
| `uninstall-samourai.sh` | `scripts/uninstall-samourai.sh` | Remove files previously installed by Samourai from a target project |

## Required Tools

These commands must be available for local install/uninstall:

| Tool | Used by | Why |
|------|---------|-----|
| `bash` | both scripts | Shell interpreter; requires Bash features such as arrays and strict mode |
| `git` | install | Verify that the target directory is inside a Git repository |
| `find` | both scripts | Discover canonical install files and detect remaining dedicated Samourai directories during uninstall |
| `awk` | both scripts | Deduplicate manifests, transform generated VS Code prompt content, and read checksum/overwrite manifests |
| `sed` | install | Normalize relative paths while collecting source files |
| `grep` | both scripts | Validate manifest entries and check existing ignore rules |
| `sort` | both scripts | Produce deterministic manifests and cleanup order |
| `sha256sum` | both scripts | Record hashes for installed files and avoid deleting installed files modified after installation |
| `cp` | install | Copy adapter files and rendered generated files |
| `mkdir` | both scripts | Create target directories and install state directories |
| `rm` | uninstall | Remove files listed in the install manifest and dedicated Samourai directories after confirmation/force |
| `rmdir` | uninstall | Remove empty managed directories after uninstall |
| `dirname` | both scripts | Resolve parent directories for copied and removed files |
| `pwd` | both scripts | Resolve source and target paths |
| `printf` | both scripts | Structured logs and generated file output |

On Ubuntu, these are provided by the base system packages plus `git`.

```bash
sudo apt-get update
sudo apt-get install -y bash coreutils findutils grep sed gawk git
```

## Optional Tools

| Tool | Used by | Why |
|------|---------|-----|
| `curl` | install | Install OpenCode automatically when the OpenCode adapter is selected and `--skip-opencode` is not used |
| `opencode` | target project runtime | Run OpenCode agents and commands after installation |
| `code` / VS Code | target project runtime | Use generated GitHub Copilot agents, prompts, skills, and MCP config |
| `docker` | target project runtime | Run MCP servers configured in `.vscode/mcp.json`, such as the GitHub MCP server |
| `node` | development validation | Validate JSON/JSONC or run repo-local checks when needed |

## Editor Runtime Notes

OpenCode installation writes:

- `.opencode/opencode.jsonc`
- `.opencode/agent/*.md`
- `.opencode/command/*.md`
- `.opencode/skills/*`

VS Code installation writes:

- `.github/copilot-instructions.md`
- `.github/agents/*.agent.md`
- `.github/prompts/*.prompt.md`
- `.github/skills/*`
- `.vscode/extensions.json`
- `.vscode/mcp.json`
- `.vscode/settings.json`

Reusable governance, templates, and decisions are installed under `.samourai/core/**`.
They are not copied into target project documentation folders.

`.samourai/AGENTS.md` is not installed by the devkit. It is generated by `/bootstrap` from the target project context. A root `AGENTS.md` compatibility entrypoint may also be generated for tools that auto-discover that filename.

## Uninstall Safety Notes

The uninstall script primarily removes files recorded in `.samourai/install/installed-files.txt`.
For installs created by recent versions, `.samourai/install/installed-files.sha256`
stores hashes for added files. If a file was changed after installation, uninstall
keeps it and prints a warning instead of deleting local work.

Dedicated Samourai/AI directories can be removed after confirmation, or with
`--force`, except project-owned artifacts under `.samourai/docai/` and
`.samourai/AGENTS.md`, which are preserved unless they were explicitly listed
in the install manifest:

- `.opencode`
- `.samurai` legacy spelling
- `.samourai/tmpai`

Shared project/editor directories are never removed with `rm -rf`:

- `.github`
- `.vscode`
- `.ai`

For `.github` and `.vscode`, uninstall removes only manifest-listed files and
then prunes empty subdirectories. For legacy root `.ai`, uninstall removes only
known Samourai files such as `.ai/local/pm-context.yaml` and then prunes `.ai`
only if it becomes empty. For the current `.samourai/ai` layout, uninstall
removes local state under `.samourai/ai/local/` but preserves project-owned
configuration under `.samourai/ai/agent/`, `.samourai/ai/rules/`, and
`.samourai/ai/context/`.
